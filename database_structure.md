# Safeway EMS Database Structure

This document outlines the database schema for the Safeway Enquiry Management System (EMS). Each database model is presented in its own table with details on field names, data types, constraints, descriptions, and example values.

---

## 1. Employee Table
Stores user accounts for Admins and Technicians.

| Column | Type | Constraints / Attributes | Description | Example |
| :--- | :--- | :--- | :--- | :--- |
| **id** | String | `@id`, `@default(uuid())` | Unique identifier for the account | `d4b8e3a2-7f12-4c8d-b9ea-b65f3fcd84c6` |
| **mobileNumber** | String | `@unique`, Required | Mobile number used for logging in | `9876543210` |
| **passwordHash** | String | Required | Bcrypt hash of user's password | `$2a$10$X7jX8L2...` |
| **role** | String | Required | User role: `"ADMIN"` or `"TECHNICIAN"` | `ADMIN` |
| **fullName** | String | Optional | Full name of the employee | `John Doe` |
| **contactPhone** | String | Optional | Contact number of the employee | `9111111111` |
| **employeeNumber**| String | `@unique`, Optional | Unique employee code | `E001` |
| **email** | String | Optional | Email ID of the employee | `john.doe@safeway.com` |
| **isActive** | Boolean | `@default(true)` | Account active status | `true` |
| **createdAt** | DateTime | `@default(now())` | Timestamp of account creation | `2026-07-14T10:00:00Z` |
| **updatedAt** | DateTime | `@updatedAt` | Timestamp of last update | `2026-07-14T10:45:00Z` |
| **deletedAt** | DateTime | Optional | Timestamp for soft deletion | `null` |

## 3. Ticket Table
Stores enquiries and jobs, along with lifecycle stages and scheduling parameters.

| Column | Type | Constraints / Attributes | Description | Example |
| :--- | :--- | :--- | :--- | :--- |
| **id** | String | `@id`, `@default(uuid())` | Unique identifier for the ticket | `a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d` |
| **ticketNumber** | String | `@unique`, Required | Autogenerated unique reference code | `EQ001` |
| **customerId** | String | Optional, `ForeignKey(Customer.id)`| ID of the customer associated with the ticket | `f9b5c3e4-8a12-4d8b-b9f1-a75d3fcd84c2` |
| **currentStage** | String | Required | Lifecycle stage: `"ENQUIRY"`, `"REFILLING"`, `"SERVICES"`, `"COMPLETED"` | `ENQUIRY` |
| **currentStatus** | String | Required | Status within the stage (e.g. `"Order Delivered"`, `"Pending"`) | `Order Delivered` |
| **itemDescription**| String | Optional | Description of cylinder/equipment | `ABC Fire Extinguisher 9kg` |
| **serialNumber** | String | Optional | Unique cylinder cylinder tag/serial number | `CYL-99823` |
| **capacity** | String | Optional | Cylinder capacity (e.g. `"2 Kg"`, `"9 Kg"`) | `9 Kg` |
| **extinguisherType**| String| Optional | Extinguisher type (e.g. `"CO2"`, `"DCP"`, `"Water"`) | `DCP` |
| **requirementCategory**| String| Optional | Category of work (e.g. `"CCTV"`, `"Refilling"`) | `Refilling` |
| **enquirySource** | String | Optional | Enquiry origin source (e.g. `"Social Media"`, `"Walk-in"`) | `Social Media` |
| **requirementDetails**| String| Optional | Notes describing customer requirement | `Refill 4 DCP cylinders` |
| **requestedDeliveryDate**| DateTime| Optional | Requested delivery date by customer | `2026-07-20T10:00:00Z` |
| **followUpDate** | DateTime | Optional | Scheduled enquiry follow-up date | `2026-07-16T12:00:00Z` |
| **latestFollowUpNotes**| String| Optional | Notes from the latest follow-up call | `Client wants a discount` |
| **deliveredDate** | DateTime | Optional | Actual date cylinder was delivered | `2026-07-15T15:30:00Z` |
| **amcYears** | Int | Optional | Number of AMC coverage years (1 to 10) | `2` |
| **amcDate** | DateTime | Optional | Calculated AMC end date (deliveredDate + amcYears) | `2028-07-15T15:30:00Z` |
| **scheduledVisitDate**| DateTime| Optional| Scheduled visit date for technician | `2026-07-15T09:00:00Z` |
| **adminNotes** | String | Optional | Service instructions logged by Admin | `Check safety seal on-site` |
| **technicianNotes**| String | Optional | Service notes logged by Technician | `Valve was jammed, replaced it` |
| **locationCoordinates**| String| Optional | Map coordinates or location address details | `https://maps.google.com/?q=12.97,80.21` |
| **assignmentType** | String | Optional, `@default("DELIVERY")` | Assignment type (e.g. `"DELIVERY"`, `"REPAIR"`) | `DELIVERY` |
| **stageData** | String | Optional | Serialized stage-specific dynamic input JSON | `{"pressureTestPassed":true,"refillMediumQty":4.5}` |
| **createdAt** | DateTime | `@default(now())` | Timestamp of ticket creation | `2026-07-14T10:00:00Z` |
| **updatedAt** | DateTime | `@updatedAt` | Timestamp of last update | `2026-07-14T10:45:00Z` |
| **deletedAt** | DateTime | Optional | Timestamp for soft deletion | `null` |

## 4. TicketFollowUp Table
Stores notes for customer follow-up calls related to enquiries.

| Column | Type | Constraints / Attributes | Description | Example |
| :--- | :--- | :--- | :--- | :--- |
| **id** | String | `@id`, `@default(uuid())` | Unique identifier for follow-up entry | `c2b3a4d5-e6f7-7a8b-9c0d-1e2f3a4b5c6d` |
| **ticketId** | String | Required, `ForeignKey(Ticket.id)` with cascade delete | Associated ticket ID | `a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d` |
| **remarks** | String | Required | Notes from follow-up conversation | `Called client, they asked to callback tomorrow` |
| **createdAt** | DateTime | `@default(now())` | Timestamp of follow-up record | `2026-07-14T11:00:00Z` |
| **deletedAt** | DateTime | Optional | Timestamp for soft deletion | `null` |

## 5. TicketAssignment Table
Stores technician job assignments.

| Column | Type | Constraints / Attributes | Description | Example |
| :--- | :--- | :--- | :--- | :--- |
| **id** | String | `@id`, `@default(uuid())` | Unique identifier for the assignment | `z1y2x3w4-v5u6-7t8s-9r0q-1p2o3n4m5l6k` |
| **ticketId** | String | Required, `ForeignKey(Ticket.id)` with cascade delete | Associated ticket ID | `a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d` |
| **employeeId** | String | Required, `ForeignKey(Employee.id)`| Assigned employee's user ID | `d4b8e3a2-7f12-4c8d-b9ea-b65f3fcd84c6` |
| **assignedAt** | DateTime | `@default(now())` | Timestamp of assignment | `2026-07-14T10:15:00Z` |
| **completedAt** | DateTime | Optional | Timestamp when technician finished job | `2026-07-14T14:30:00Z` |
| **status** | String | Required | Status of assignment: `"ASSIGNED"` or `"COMPLETED"` | `ASSIGNED` |
| **notes** | String | Optional | Post-visit technician notes | `No one was at the home, returned.` |
| **deletedAt** | DateTime | Optional | Soft-delete timestamp for inactive assignments | `2026-07-14T10:20:00Z` |

*Note: There is a unique constraint on `[ticketId, employeeId]` preventing duplicate assignments.*

## 6. TicketHistory Table
Stores audit trails of job status and stage transitions.

| Column | Type | Constraints / Attributes | Description | Example |
| :--- | :--- | :--- | :--- | :--- |
| **id** | String | `@id`, `@default(uuid())` | Unique identifier for history entry | `h1i2j3k4-l5m6-7n8o-9p0q-1r2s3t4u5v6w` |
| **ticketId** | String | Required, `ForeignKey(Ticket.id)` with cascade delete | Associated ticket ID | `a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d` |
| **changedById** | String | Required, `ForeignKey(Employee.id)` | User ID who made the changes | `d4b8e3a2-7f12-4c8d-b9ea-b65f3fcd84c6` |
| **fromStage** | String | Optional | Stage before change | `ENQUIRY` |
| **toStage** | String | Optional | Stage after change | `REFILLING` |
| **fromStatus** | String | Optional | Status before change | `Order Delivered` |
| **toStatus** | String | Optional | Status after change | `Refilling Order Received` |
| **remarks** | String | Optional | Summary notes explaining changes | `Bulk transitioned stage from ENQUIRY to REFILLING` |
| **createdAt** | DateTime | `@default(now())` | Timestamp of status modification | `2026-07-14T10:45:00Z` |
| **deletedAt** | DateTime | Optional | Timestamp for soft deletion | `null` |
